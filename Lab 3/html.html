¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet son elementos de interconexión, servidores y puestos finales, el medio. Y la WWW es el conjunto de páginas web conectadas mediante enlaces dirigidos, la red.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET El método GET  solicita una representación de un recurso específico, sólo deben recuperar datos.
HEAD El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
POST El método POST se utiliza para enviar una entidad a un recurso, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
PUT El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
DELETE El método DELETE borra un recurso en específico.
PATCH El método PATCH  es utilizado para aplicar modificaciones parciales a un recurso.

Sistemas de información

Investigación 1: Ciclo de vida y de desarrollo de los sistemas de información

El ciclo de vida de un sistema de información, son las etapas por las que pasa un sistema especifico y nos permite recopilar, transmitir, almacenar, visualizar y procesar información.

El ciclo de vida de un sistema de información consta de 7 etapas y son representadas por la planificación, el analisis, el diseño, la implementación, las pruebas, la instalación y el uso y mantenimiento.

La planificación, es una de las etapas más importantes durante el ciclo de vida de un sistema de información, ya que de este dependerá en gran parte el éxito del mismo, sin dejar a lado todas las demás etapas que tienen gran importancia, la planificación consiste en una serie de tareas previas con el fin de tener un objetivo claro.

Los requerimientos de un sistema de información son especificados en la etapa del análisis, en esta etapa también se establece qué hará el sistema y qué realmente se necesita realizar.

Dentro de la etapa del diseño, se establecen las diferentes alternativas para llegar al objetivo, descartando algunas y estableciendo la que pueda traer mayores beneficios al sistema de información.

Después de diseñar la mejor alternativa, sigue la etapa de la implementación en la cual se eligen las herramientas adecuadas dependiendo del sistema de información que se desee construir, estas herramientas incluyen el lenguaje de programación, sin embargo, todo se basará en las desiciones tomadas en las etapas anteriores.

En la etapa de las pruebas, se podrán detectar y corregir los posibles errores de las etapas anteriores, en esta etapa nos daremos cuenta si es factible que el sistema de información cumpla con su objetivo.

En la instalación, se decide como se colocará el sistema de información para tener mayor eficiencia y eficacia, tanto el hardware como el software.

El uso y mantenimiento, es la etapa que más tiempo consume en el ciclo de vida de los sistemas de información, ya que debido a que vivimos en una sociedad cambiante, el mantenimiento tiene que ser constante para poder adaptar el sistema a las nuevas necesidades.

Todas las etapas dentro del ciclo de vida de los sistemas de información, son de gran importancia, ya que sin uno, no se puede realizar el otro, al juntar todas estas etapas podremos obtener un sistema útil, que nos permita realizar diferentes funciones con un sistema de información especifico.

Descripciones

URL

Uniform Resource Locutor (URL), es una secuencia de caracteres que nos permite identificar recursos dentro del Internet para que puedan ser utilizados.

MIME

Multipurpose Internet Mail Extensions (MIME) es un tipo estandarizado que nos permite indicar el formato original de un documento.

Métodos y códigos de errores HTTP

Los códigos de errores y métodos HTTP, describen de forma abreviada la respuesta HTTP. Los errores, dependiendo del número, pueden ser ocasionados por diferentes factores:

  • 1XX Respuestas informativas
  • 2XX Peticiones correctas
  • 3XX Redirecciones
  • 4XX Errores del cliente
  • 5XX Errores de servidor
